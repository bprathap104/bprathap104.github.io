Cloud Custodian - Learning

Nov 12th, 2020 1:02 am

Cloud Custodian is a policy based cloud security tool. It uses serverless capabilities of each of the 3 Cloud Providers to manage its resources to be compliant with Cloud security policies.

The serverless capabilities of each of the 3 service providers are:

1) AWS - Lambda

2) GCP - Cloud Functions

3) Azure - Azure Funcitons + Logic App

By default Custodian uses YAML as the input file for policies while it converts it into JSON internally.

We can first start understanding the AWS code. Then we can move into Azure & GCP

Each of the above code base can be logically separated into two logical codes

1) One set of code for custodian CLI

2) Other set of code that actually enforces compliance on the resource
